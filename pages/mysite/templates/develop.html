Develop
NabiTech is a firm believer of open-sourcing. Every program that makes Nabi Cobot itself is a testament of this, albeit some of the work that are unstable or poses danger are close-sourced. Nabi Local Server is a highly extensible application that isn’t built on top of any framework but a locally hosted Django server which is ran using terminal scripts. This eliminates any build dependencies at a cost of a few installation requirements which is documented. It is modular, you can add any component based on your use case. The core idea of this server is to connect multiple Nabi Cobots in your workspace and control them in a single server. With time, we hope to implement this as a single module installable with PyPi.
Running the Code
Your Nabi Cobot is equipped with the right dependencies to run most of the code. However, if you intend to perform a fresh installation, you can complete these steps.
Clone the GitHub repository.
git clone https://github.com/lukasdante/nabi-dialogflow
Create a virtual environment using conda or Python 3. Installation instructions for conda are found here. In this documentation, we will be using Python’s virtual environment.
python -m venv env # you can change env to preferred name
Activate your virtual environment
source env/bin/activate
In your terminal, use pip to install all necessary requirements for running the code.
pip install -r requirements.txt
You may need to install these build dependencies first.
sudo apt-get install cmake build-essential libasound2-dev
sudo apt-get update
Once that you have installed these dependencies, or this isn’t a fresh installation, you can simply run the server.
cd pages
python manage.py runserver
If you followed the steps correctly, a local host network will be provided like this.
http://127.0.0.1:8000/
